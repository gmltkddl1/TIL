# 구글 드라이브 설계
## 기본 구조
- 클라이언트
- 파일 블록서버: 파일을 분할하고 압축하고 암호화해서 클라우드에 전송, 유저의 다운로드 요청시 파일 제공
- 클라우드 저장소 (S3)
- 알림서버: 클라이언트와 롱폴링 연결을 유지하여 변경사항을 알려준다
- API 서버: 파일 업로드와 다운로드를 제외한 모든 요청 처리
- 메타 데이터 저장소
## 충돌해결
- file의 버전을 이용하여 기존에 조회햇던 버전와 다를 경우 충돌로 인식하여 볃합처리
## 네트워크 대역폭 절약
- 블록으로 나눠진 파일의 해쉬값을 비교하여 변경이 있는 부분만 클라우드로 업로드
- 압축 적용
## 저장소 용량 절약
- 최대 보관 버전의 숫자를 제한하거나 중요한 버전만 보관
- 미사용 파일 아카이브에 보관
