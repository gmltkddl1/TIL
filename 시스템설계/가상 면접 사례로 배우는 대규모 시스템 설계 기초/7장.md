# 유일 ID 생성기

## 요구사항
- ID는 유일해야한다.
- 아이디는 숫자로만 이루어져야한다.
- 아이디는 64비트여야한다.
- 아이디는 시간순 정령이 가능해야한다
- 아이디는 초당 10000개 생성이 가능해야한다.

## 방범
1. 다중 마스터 복제
- 각 서버에서 auto_increment로 ID를 생성한다. 증가값은 서버가 N대라고 할때 N씩 증가 시킨다. 초기 값을 1~N까지 부여하면 중복 없이 아이디 생성 가능
  - 서버 추가 삭제의 어려움
  - 정렬 불가
  - 지리적 편향을 조절하기 어려움 한국에서만 엄청쓰면 특정 나머지의 아이디만 폭증
2. UUID
- 중복가능성이 매우 낮음 알고리즘으로 128비트의 아이디를 생성함
  - 장점
  - 생성이 간단함
  - 서버에서 동기화 없이 실행가능하므로 확장이 쉬움 
  - 단점
  - 시간순 정렬 불가능
  - 크기가 128임
  - 숫자가 아닌 값이 있다.
3. 티켓서버
- 중앙집중형 auto_increment 서버에서 아이디를 생성한다
  - 장점
  - 간단하다 
  - 단점
  - 단일 실패지점이 된다
  - 이를 피하기 위해 다중화하면 동기화 이슈 발생
4. Snowflake
- 41비트 타임스탬프에 5비트 데이터센터 5비트 서버 12비트 일련번호를 부여하는 아이디를 생성한다.
- 요구사항 모두를 만족하지만 특정 기간이 지나면 사용이 불가하다. 버전을 추가해서 해결 가능
