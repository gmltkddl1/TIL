# 뉴스피드
## 기본 구조
### 피드 생성
피드는 기본적으로 손실되어도 서비스 운영에 문제가 없으므로 캐시에 저장하는거 같다.
- 사용자
- 인증서버
- 피드 생성 서비스
- 피드 전송 서비스
  - 피드 생성시 팔로워들의 캐시에 피드 ID를 넣음 ex) gmltkddl1 : [feedid1, feedid2 ....]
  - 팔로워가 일정 숫자 이상인 유저는 위 과정을 실행하지 않고 읽을때 가져옴
- 피드 캐시
### 피드 읽기
- 사용자
- 인증서버
- 피드 읽기 서버
  - 기본적으로 자신의 피드 캐시를 읽어서 가져옴
  - 유명인사의 데이터의 최신 데이터를 가져와서 합쳐서 보여줌
